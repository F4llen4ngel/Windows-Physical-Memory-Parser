257 * The address bits in the above structures should be
258 * right. You can also use the previously-mentioned
259 * Windows-specific general page table structure definition,
260 * which I have taken out of KD and added a definition
261 * for below.
262 *
263 * 1: kd> dt ntkrnlmp!_MMPTE_HARDWARE
264 * +0x000 Valid : Pos 0, 1 Bit
265 * +0x000 Dirty1 : Pos 1, 1 Bit
266 * +0x000 Owner : Pos 2, 1 Bit
267 * +0x000 WriteThrough : Pos 3, 1 Bit
268 * +0x000 CacheDisable : Pos 4, 1 Bit
269 * +0x000 Accessed : Pos 5, 1 Bit
270 * +0x000 Dirty : Pos 6, 1 Bit
271 * +0x000 LargePage : Pos 7, 1 Bit
272 * +0x000 Global : Pos 8, 1 Bit
273 * +0x000 CopyOnWrite : Pos 9, 1 Bit
274 * +0x000 Unused : Pos 10, 1 Bit
275 * +0x000 Write : Pos 11, 1 Bit
276 * +0x000 PageFrameNumber : Pos 12, 36 Bits
277 * +0x000 ReservedForHardware : Pos 48, 4 Bits
278 * +0x000 ReservedForSoftware : Pos 52, 4 Bits
279 * +0x000 WsleAge : Pos 56, 4 Bits
280 * +0x000 WsleProtection : Pos 60, 3 Bits
281 * +0x000 NoExecute : Pos 63, 1 Bit
282 */
